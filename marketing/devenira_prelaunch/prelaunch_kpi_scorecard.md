# Prelaunch KPI Scorecard

Goal:

Measure whether the message is working before spending real money.

## Primary Metrics

### Traffic
- landing visits
- visits by source
- repeat visits

### Interest
- hero CTA clicks
- total beta CTA clicks
- CTA click-through rate

### Activation
- try page visits
- guest check-ins started
- guest check-ins completed
- account creation clicks from try result

### Learning
- replies to outreach
- interview bookings
- content saves
- content replies / DMs

## Minimum Dashboard

| Metric | Weekly target | Why it matters |
|---|---:|---|
| Landing visits | 100+ | Enough volume to compare messages |
| Beta CTA CTR | 8%+ | Tests whether promise is clear |
| Try-page conversion from landing | 5%+ | Confirms offer is believable |
| Guest check-in completion rate | 35%+ | Tests whether flow is usable |
| Register CTA clicks from try result | 15%+ | Tests whether weekly loop feels worth saving |
| Outreach replies | 20%+ | Tests whether problem statement resonates |
| User interviews | 5 per week | Keeps strategy tied to reality |

## Weekly Review Template

### 1. Acquisition
- Where did visitors come from?
- Which posts or DMs caused visits?

### 2. Conversion
- Which CTA got clicked most?
- Which section lost attention?

### 3. Activation
- Did people start the check-in?
- Where did they abandon the try flow?

### 4. Qualitative Learning
- What wording did users repeat back?
- What objection came up most often?

### 5. Next Decision
- keep headline
- change subhead
- simplify CTA
- add more trust copy
- improve try flow copy

## Stoplight Rules

### Green
- people understand the product fast
- CTA click rate is healthy
- interviews confirm the problem

Action:
- keep message stable
- publish more content
- expand outreach volume

### Yellow
- traffic arrives but CTA clicks are weak
- people click but do not finish try flow

Action:
- simplify landing headline
- reduce copy weight above the fold
- clarify privacy and beta expectations

### Red
- people do not understand what the product is
- interviews reveal weak problem resonance
- people ask for unrelated features immediately

Action:
- narrow ICP
- rewrite positioning
- stop adding more features to the message

## Daily Founder Loop
- check visits
- check CTA clicks
- check replies / DMs
- log one insight

## Weekly Founder Loop
- update the table
- choose one message to keep
- choose one weak point to fix
- decide what to publish next week

## Hard Rule
Do not use paid ads seriously until these are true:

- the landing headline is stable
- the CTA is stable
- the try flow has acceptable completion
- at least 10 user conversations confirm the problem is real
